Classification Trees¶

Decision trees are a tree-shaped set of questions (like the game 20 Questions) that can tell you which class an example falls under. They can also be used for regression:

  • Regression trees when the outcome is continuous
  • Classification trees when the outcome is discrete (falls into a finite number of groups)

Consider a case where we have three different classes of wine and we are tasked to classify them based on their alcohol percentage and malic acid level. As seen in the figure below, we have 178 data points of three different wine types (class_0, class_1, and class_2), which are plotted based on their alcohol percentage and malic acid level. The plot shows that:

  • class_0 wines tend to have a higher alcohol content, and lower malic acid content than the others
  • class_1 is characterized by lower levels of both alcohol and malic acid
  • class_2 tends to have higher levels of both

We can also see that there is no clear separation between the classes in the plot, meaning that these characteristics overlap somewhat.

Figure 1. Three classes of wines based on their Alcohol and Malic Acid level

One simple approach to do this classification task is to find some binary rules based on alcohol percentage and malic acid level which separate different types of wine. In the figure above, you can observe alcohol percentage for class_1 wine is typically less than 12.7 percent (58 out of 72), whereas alcohol percentage for the other two classes tends to be higher. So, we can select the alcohol level as our splitting feature and 12.7 as our threshold (splitting point). By selecting 12.7 as our threshold, we create two disjoint subsets of data. The figure below demonstrates two new regions based on separation by alcohol level.

Figure 2. Binary Splitting based on Alcohol Level

As seen in the figure above, in the region on the right (alcohol level higher than 12.7%), there is no clear distinction among class_0 or class_2 wines. However, if we look closely, we can observe that splitting the data set based on malic acid level can give us a better distinction between these two classes of wines. So, in our next step we choose malic acid level as our splitting feature and 2.3 as our threshold. We will end up with three regions which now are fairly distinct per class (visualized by color). The result is that 50 data points of class_0 out of 59 have a less than 2.3 malic acid level and 42 data points of class_2 out of 48 have more than 2.3 as their malic acid level, as visualized below.

Figure 3. Binary Splitting based on Malic Acid Level

You can observe a sample tree based on the rules which we found so far, visualized below. Each node has the following attributes based on the training set:

  • Binary rule: Decision factor for traversing over the tree
  • Entropy: A value measuring the impurity of each node (we will discuss in more detail later in this module)
  • Samples: The number of examples in each region
  • Value: Number of instances for each class
  • Class: Predicted class for each node

Figure 4. Set of binary rules for classifying wine data points

Now, given a sample wine data point, we would like to predict the class of the wine. The process starts from the root and we apply the binary rule at each node as we traverse the tree. When we get to a leaf of the tree, we can calculate the probability of that the instance belongs to any of the possible classes.

For example, if the value of the leaf containing our sample wine is $[50, 7, 4]$, then we calculate the affiliation of the sample to each class as follows:

  • $P[\text{sample wine belongs to class_0}] = \frac{50}{50+7+4} = \frac{50}{61} = 0.81$

  • $P[\text{sample wine belongs to class_1}] = \frac{7}{50+7+4} = \frac{7}{61} = 0.11$

  • $P[\text{sample wine belongs to class_2}] = \frac{4}{50+7+4} = \frac{4}{61} = 0.06$

As you can see classification trees are very interpretable and are close to the way humans make decisions.

Impurity Measures¶

An impurity measure is a measurement of the homogeneity or heterogeneity of a collection of items. Impurity measures are a fundamental part of decision trees algorithms. You can think of proceeding down the tree as a process of purification: at each step we are applying an additional filter to separate the classes further.

Consider the case where we have three bags containing ten marbles each. The first bag has ten blue marbles, the second bag has eight blue marbles and the rest are red. The third bag contains half and half, blue and red marbles.

The first bag is considered as completely pure, since there is no mixture of blue and red marbles. Intuitively, the second bag is more pure than the third since the majority of the bag is from the same class (color). Gini, entropy and misclassifications are alternative ways to measure impurity in a dataset. We will discuss Gini and entropy in this module.

Gini Impurity¶

Gini impurity is a measure of how often a randomly chosen element from the set would be incorrectly labeled if it were randomly labeled according to the distribution of labels in the subset. The Gini impurity can be computed by summing the probability $p_{i}$ of an item with label $i$ being chosen times the probability ${\displaystyle \sum _{k\neq i}p_{k}=1-p_{i}}$ of a mistake in categorizing that item. It reaches its minimum (zero) when all cases in a tree node fall into a single target category.

To compute the Gini impurity for a set of items with ${\displaystyle J}$ classes, suppose ${\displaystyle i\in \{1,2,...,J\}} $, and let $p_{i}$ be the fraction of items labeled with class ${\displaystyle i}$ in the set.

$$ {\displaystyle \operatorname {I} _{G}(p)=\sum _{i=1}^{J}p_{i}\sum _{k\neq i}p_{k}=\sum _{i=1}^{J}p_{i}(1-p_{i})=\sum _{i=1}^{J}(p_{i}-{p_{i}}^{2})=\sum _{i=1}^{J}p_{i}-\sum _{i=1}^{J}{p_{i}}^{2}=1-\sum _{i=1}^{J}{p_{i}}^{2}} $$

We will try to measure the Gini impurity (or Gini index) for the marble bags example we discussed above.

bag_1 = ["blue", "blue", "blue", "blue", "blue", "blue", "blue", "blue", "blue", "blue"]

bag_3 = ["red", "red", "red", "red", "red", "blue", "blue", "blue", "blue", "blue"]

# Counting the number of marbles with the same color
def class_counts(rows):
    counts = {}  
    for row in rows:
        label = row[-1]
        if label not in counts:
            counts[label] = 0
        counts[label] += 1
    return counts

# Calculate Gini Index for each bag  
def gini(rows):
    counts = class_counts(rows)
    impurity = 0
    for lbl in counts:
        prob_of_lbl = counts[lbl] / float(len(rows))
        impurity += prob_of_lbl * (1 - prob_of_lbl)
    return impurity

Entropy Impurity¶

Intuitively, the entropy of a discrete random variable $X$ is a measure of the amount of uncertainty associated with the value of $X$ when only its distribution is known. The measure of entropy associated with each possible data value is the negative logarithm of the probability mass function for the value. Following from this, we can find the equation for it as follows where $p_{i}$ is the fraction of items labeled with class ${\displaystyle i}$ in the set.

$$S=-\sum _{i}p_{i}\log {p_{i}}$$
In [0]:
# Add Numpy package since we have numerical processing   
import numpy as np

# Calculate entropy for each bag 
def entropy(rows):
    counts = class_counts(rows)
    impurity = 0
    # lbl represents assigned label to the marbles 
    for lbl in counts:
        prob_of_lbl = counts[lbl] / float(len(rows))
        impurity += -prob_of_lbl * np.log2(prob_of_lbl) 
    return impurity

Learning Classification Trees¶

The most well-known algorithms for learning classification trees are a family of algorithms known as CART (Classification and Regression Trees), ID3, C4.5 and C5. In this module, we will discuss the CART algorithm. The CART algorithm grows a binary tree in the same way we did in the previous section. CART seeks to minimize the impurity in each leaf, and build regions with more distinctive classes.

In the previous section, we identified the splitting features and thresholds by observing the scatter plot. Although it is an intuitive way to begin, CART obviously cannot use the same approach. CART will search over all features and values to find a feature ($f$) and threshold ($t_i$) which produces the purest regions. The cost function for these possible splits is as follows:

$$\text{Cost}(k,t_i) = \frac{m_{left}}{m}G_{left} + \frac{m_{right}}{m}G_{right} $$
  • $m_{left}$ is the number of samples on the left node (same for $m_{right}$)
  • $G_{left}$ is the impurity measure on the left node (same for $G_{right}$)
  • Feature $k$ and threshold $t_i$ are those with lowest cost

After CART splits the training set into two regions, it can recursively do the same splitting task on each subregion until it cannot reduce the impurity or it reaches a user-defined stopping criterion.

Unfortunately, finding a fully optimized tree is an NP-Complete problem and has exponential complexity. Instead, we can use a greedy algorithm which finds the best pair of ($f, t_i$) at each iteration. Greedy algorithms do not always give the best possible solution but are fast and simple. There could be a tree with better accuracy but greedy algorithms often give a good enough solution. So the CART algorithm is the combination of using a greedy algorithm with an impurity measure.

CART will perform the following steps:

  1. Start at the root node as the parent node

  2. Find a feature and threshold to minimize the sum of the child node impurities

  3. Generate two disjoint regions based on the selected feature and threshold and assign them to child nodes

  4. Stop when leaves are pure or early stopping criteria are satisfied, or else repeat steps 1 and 2 for each new child node

The following are a few other early stop criteria that may be used depending on the specific CART algorithm implementation:

  • The maximum depth of the tree
  • The minimum number of samples required to split an internal node
  • A node will be split if this split induces a decrease of the impurity greater than or equal to this value
  • If a threshold is set for early stopping in tree growth:
    • A node will split if its impurity is above the threshold, otherwise it remains a leaf
    • A node will be split if this split induces a decrease of the impurity greater than or equal to this value

Given a trained classification tree model and a new data point $X$, the process of producing a prediction will go through the following steps to estimate $X$:

  1. At each node, a feature is compared to a threshold $X_i \leq t_i$ ($t_i$ is the specified threshold in the node)

  2. If $X_i \leq t_i$ is true, and the node is not a leaf, then the example is sent to the left child of the node, otherwise to the right

  3. This continues until a leaf is reached

  4. The leaf contains a distribution across the classes, based on the training data

  5. The predicted class for $X$ is the class with highest probability in the leaf

Decision Tree Disadvantages¶

Decision trees are quite powerful solutions for classification and regression problems. However, they have some shortcomings, specifically with numerical features.

Overfitting¶

One of the problems with decision trees is they tend to overfit very easily. In the figure below, you can see the decision tree is overfitted to anomalies. Instead of two clearly defined zones on the plot, we see the zones jumbled together, especially in the overlap region and around outliers. Since decision trees are non-parametric, they make very few assumptions about the data and will perfectly fit to the data if allowed to grow until every leaf has only one training sample left in it. We need to always remember that real data includes noise along with the signal. We want our algorithms to learn the signal. However, when they are allowed to have too much power (i.e. capture too much complexity in the data and become overfitted), they will learn the noise as well.

Figure 5. Overfitting Problem With Decision Tree

One way to avoid overfitting is to put some constraints on the tree during the training process. We can restrict the shape of the tree by using early termination criteria. In the figure below, you can observe how min_sample_leaf was used to avoid overfitting.

Figure 6. Growing the Decision Tree by Regularization

Another effective regularization method is max_depth. By restricting the depth of the tree we can avoid the tree growing until every sample has its own leaf.

Instability¶

It is important to be aware that decision trees are also very sensitive to small changes in the data. Small changes not only change the prediction, they could also change the whole structure of the tree drastically. To illustrate this, we will explore the Iris dataset once again. The figure below shows how a classification tree classified three types of flowers (irises) by their petal width and length. Regions in each class are separated by straight lines. Each separator line has a label associated with it which indicates where inside the binary tree they are formed (labeled as "Depth").

Figure 7. Classification with Decision Tree on Iris Dataset

If we remove the widest (iris versicolor with petal width: 4.8 cm, petal length: 1.8 cm) from the training set the classification tree changes drastically, as you can observe in the figure below.

Figure 8. Showing Instability by Moving Widest Data Point

NOTE: There is a way to address instability. Random forests can limit instability by averaging among multiple trees. We will learn more about random forests later.

Decision Boundaries are parallel to the axis¶

As we've seen, an important characteristic of decision trees is that they are built based on comparing feature values to thresholds. Because of this, the decision boundary is always formed by line segments parallel to the axes of the features. This means, in particular, that the performance of the algorithm is not invariant to transformations of the data, such as rotations.

The figure below demonstrates what can happen when data rotation occurs. The right plot shows a dataset which is fairly linearly separable. When the data is rotated 45 degrees this doesn't remain true, since all splits would need to be perpendicular to the axis. One of the ways to solve this problem is by using one of the dimension reduction algorithms like PCA which may result in better orientation of the data.

Figure 9. Decision Boundary Changes by Rotating Data

Regression Trees¶

Regression trees are decision trees where the target variable is continuous. Regression trees are the same as classification trees in most ways, but there are two main differences:

  1. Their predicted output is a numerical value rather than a discrete class label.

  2. Instead of using impurity measures, they use Mean Squared Error (MSE) as the cost function and are trying to decrease variance instead of impurity.

Consider the moon-shaped dataset in the figure below. We would like to build a regression model which predicts $y$ based on the $x$ value.

Figure 10. Moon Shaped Data Set

One crude predictive technique we could use, if we didn't know any better methods, would be to separate the $x$ values into ten percentiles and calculate the average for each bucket (a kind of weighted nearest neighbours). In the figure below, the blue lines show the average value for each bucket. The length of the line is the range of $x$ values.

Figure 11. Calculated Average for each Percentile

However, the bucketing approach based on percentiles only works for datasets with one dimension. Even in this case, this approach might not give us very good predictions. As you can see in the figure above, in the middle of the plot there is less variance in each bucket than there is at the edges of the plot.

Rather than bucketing, we can use the binary tree approach again, but instead of trying for decreasing impurity, we work to decrease variance and the output returns the mean of each region.

Figure 12. Sample Regression Tree on Moon Dataset

In the figure above, the root node threshold is 0.79 since it gives the smallest accumulative variance for both the right and left regions. The next thresholds are 0.15 and 0.89.

Figure 13. Binary Tree on Moon Shape Dataset

The way we estimate the values in regression trees is quite similar to the way we did it for classification trees. For each value we traverse the tree until we find the leaf with minimum MSE. After finding the most relevant leaf, the value attribute is the average of all samples that exist in the leaf. For each leaf, the "samples" value represents the number of instances in the leaf.

Learning Regression Trees¶

Growing regression tress is not that different from growing classification trees as they both are built by doing successive binary partitions (based on the different predictors). The only difference is instead of using Gini or entropy, variance will now be minimized between samples in the region. The following is the cost function that regression trees use:

$$\text{Cost}(k,t_k) = \frac{m_{left}}{m}\text{MSE}_{left} + \frac{m_{right}}{m}\text{MSE}_{right} $$

$\text{MSE}$ of a node is defined as:

$$\text{MSE} = \sum_{i: node} (y^{(i)}-\hat{y}_{node})^2$$

which is the sum of the squared differences between each example and the average values.

Ensemble methods¶

In statistics and machine learning, ensemble methods use multiple learning algorithms to obtain better predictive performance than could be obtained from any of the constituent learning algorithms alone. Ensemble methods combine multiple (similar or different) machine learning classifiers and regression models to improve the results. The idea came from the wisdom of crowds, which states that large groups of people are often collectively smarter than individual experts working alone when it comes to problem solving, decision making, innovating and predicting.

Voting¶

The voting classifier is one of the ensemble learning methods which predicts a target value by using the majority vote of pre-trained classifiers. Two types of classifier exist: hard voting and soft voting. Hard voting returns the class most voted for by the classifiers, whereas soft voting averages the output probabilities from all the classifiers.

What follows is an example of a voting classifier on the moon shaped dataset. First, we generate the dataset and split the dataset into train and test groups.

Bagging¶

Bagging or bootstrap aggregate predictors are another type of ensemble learning method. They generate multiple versions of a predictor and aggregate them. Multiple versions are formed by making bootstrap replicates of the learning set and using these as new learning sets. In statistics, bootstrapping is any test or metric that relies on random sampling with replacement.

Bagging is similar to voting as it uses the same training algorithm. However, unlike voting, each classifier in the ensemble uses a different dataset. The aggregation averages over the different predictions when predicting a numerical outcome and takes the majority vote when predicting a class.

NOTE: If the ensemble algorithm does not replace the data, it's called pasting.

In the following figure, you can observe the difference between decision boundaries with and without bagging.

As seen above, the bagging algorithms have lower variance than each of the single predictors. Bagging can be a very effective ensemble for algorithms that have high variance like decision trees.

Random Forest¶

A random forest or random decision forest is an ensemble learning method for classification and regression which is built by bagging an ensemble of decision trees. Random forests are one the most popular algorithms as they are versatile and often perform well. Random forest algorithms use a different approach for learning trees. Instead of searching over all the features for each best splitting feature and threshold, at each step it randomly selects a subset of features. You might be wondering how this could possibly be an improvement. The answer is that it avoids overfitting.

In scikit-learn you'll find a RandomForestClassifier class which is convenient and optimized for random forest algorithms (and similarly RandomForestRegressor for regression tasks).

We will now present an example of using a random forest with the wine data. By using RandomForestClassifier, you can adjust most of the hyperparameters for decision trees and bagging at the same time.

Extremely Randomized Trees¶

Extremely randomized trees (extra trees) is a version of the random forest algorithm which takes randomization to the next level by randomly selecting a threshold for each feature instead of finding the best optimized threshold.

From the bias-variance point of view, the rationale behind the extra trees method is that the explicit randomization of the threshold and feature combined with ensemble averaging should be able to reduce variance more strongly than the weaker randomization schemes used by other methods. The usage of the full original learning sample rather than bootstrap replicas should minimize bias. From a computational point of view, the complexity of the tree growing procedure is, assuming balanced trees, on the order of $N \cdot log(N)$ with respect to learning sample size, like most other tree growing procedures. However, given the simplicity of the node splitting procedure we expect the constant factor to be much smaller than in other ensemble based methods with locally optimized thresholds.

Here is an example of building extra trees using scikit-learn.

Boosting¶

Boosting is an approach to machine learning based on the idea of creating a highly accurate prediction rule by combining many relatively weak and inaccurate rules. The main idea behind boosting is to train predictive models sequentially. Each new predictive model is built to reduce the error of the previous predictors.

AdaBoost¶

AdaBoost (Adaptive Boosting) is built from a sequence of weak (relatively poor) learners in which each one is tweaked in favor of those instances which are misclassified by the previous classifier. The way AdaBoost works is that each classifier (e.g. often a shallow decision tree) gets trained and makes predictions on the training set. At each step two types of weights are generated: one for the classifier and a second one for each of the data points in the training dataset. The classifier's weight correlates with how well the classifier predicts. The weights on training examples increase when they are misclassified. After each step, the results are be passed into the process of building the next classifier. By increasing the weights of misclassified data it biases the new weak classifier to try harder on the previous classifier's mistakes.

AdaBoost first gives an equal weight to all training instances. For example, if we have $m$ data points in our training data, each of these instances will have a weight of $\frac{1}{m}$. As each predictive model is trained we can calculate the error rate of the $j$th model by using following equation:

$$ r_j = \frac{\sum_{i=1, \hat{y_j}^{(i)} \ne y^{(i)}}^m w^{(i)}}{\sum_{i=1}^m w^{(i)}}$$

$r$ is summation of the current weights of misclassified instances divided by all the current weights of all instances. After calculating $r$ we can find the weight $\alpha$ for each predictor. This weight shows how well the predictor performs. The equation is:

$$ \alpha = \eta \log \frac{1 - r} {r} $$

After calculating $\alpha$, $w(i)$ will be calculated as follows:

$$ w(i) = \begin{cases} w(i) & \hat{y_i} = y_i, \\ w(i)\exp(\alpha) &\hat{y_i} \ne y_i, \end{cases} $$

After we calculate the weights for all the samples, then all the weights will be normalized by dividing by $\sum_ {i=1}^{m} w(i)$. By this operation, the weights will be increased for misclassified instances. The next algorithm will pay more attention to trying to predict a good result for these instances. For regression, the optimization will be on a weighted average of MSE.

After the training phase, the prediction is made by finding the weighted average of all the predictive models in the ensemble. The prediction equation is as follows:

$$\hat{y_i}=\sum_{j=1}^{n} \alpha_j \hat{y_j}(x_i)$$

Let's try a multiclass version of AdaBoost called SAMME (Stagewise Additive Model) using a multiclass exponential loss function. If SAMME is used for two classes, it is equivalent to AdaBoost. Here is an example of the SAMME.R algorithm using scikit-learn for wine classification.

Gradient Boosting¶

Gradient boosting is a machine learning technique for regression and classification tasks, which again produces a prediction model in the form of an ensemble of weak prediction models, typically decision trees, sequentially, with each one correcting its predecessor. The main difference between AdaBoost and Gradient Boosting is that gradient boosting, instead of modifying the weight of the training instances at each step, will try to fit a new predictor to the residual errors made by the previous predictor.

We can build a gradient boosting algorithm by repeating three steps:

  1. Learn a predictive regression model

  2. Compute the error residual

  3. Learn to predict the residual

In the following example, we will look at predicting house prices in Boston by implementing our own gradient boosting algorithm. We demonstrate what happens at each step. In the first step, we load the dataset and import the libraries we need.

Stacking¶

Stacked generalization or stacking is yet another way of creating an ensemble of multiple models. Compared to the other ensemble models we've discussed so far, it is less widely used. It most commonly is used to combine heterogeneous models. Stacked generalization works by deducing the biases of the generalizer(s) with respect to the provided learning set. The stacking algorithm first splits the data into two mutually exclusive subsets. The first subset of data is used to train a set of first-layer predictive models. Next, we use these predictive models to generate predictions given the predictors in the second dataset. Then, we train another layer of models using the predicted values from the first layer as the features for our second layer. We can repeat this process to create as many layers as we like.
